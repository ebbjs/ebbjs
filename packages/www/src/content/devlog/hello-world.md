Welcome to the ebb devlog. This is where I'll share updates on the project's development, design decisions, and lessons learned along the way.

## What is ebb?

Ebb is a framework for building local-first applications. The goal is simple: make the network optional. Your app should work offline, sync seamlessly when connected, and handle conflicts gracefully.

## Why a devlog?

Building in public keeps me accountable and helps document the journey. If you're interested in local-first development, CRDTs, or sync engines, there should be something useful here.

## What's next?

I'm currently working on the core sync protocol. Stay tuned for updates on:

- The hybrid logical clock implementation
- How we handle conflict resolution
- The permission system design

Thanks for following along.
